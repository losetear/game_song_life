# 古代生活模拟游戏 — 设计稿概述

## 游戏代号
暂定：**市井录**

## 一句话描述
一款以宋朝都城为背景的网页文字冒险游戏，玩家扮演古人体验从生到死的一生，世界由GOAP驱动的NPC自主运转，事事有回应。

## 核心设计原则
1. **事事有回应** — 任何可交互对象都有反馈，没有"死胡同"
2. **动态涌现** — NPC有自主目标与行为，事件由世界规则自然涌现，非脚本触发
3. **选择有重量** — 永久死亡机制让每个决策都有后果
4. **自由度优先** — 没有固定主线，玩家自己定义人生目标
5. **短反馈循环** — 每个游戏日内都有明确的小目标可完成，让玩家持续获得成就感

## 核心游戏循环（原稿缺失，补充）

```
                ┌──────────┐
                │  新的一天  │
                └────┬─────┘
                     ▼
              ┌──────────────┐
              │ 查看今日状况   │ ← 饥饿/疲劳/任务/消息
              └──────┬───────┘
                     ▼
           ┌───────────────────┐
           │ 规划今日行动       │ ← 有限时辰，需取舍
           │ (工作/社交/学习/探索)│
           └────────┬──────────┘
                    ▼
            ┌──────────────┐
       ┌───▶│  执行行动     │◀───┐
       │    └──────┬───────┘    │
       │           ▼            │
       │    ┌──────────────┐    │
       │    │ 获得反馈/奖励  │    │ ← 金钱/技能/关系/消息
       │    └──────┬───────┘    │
       │           ▼            │
       │    ┌──────────────┐    │
       │    │ 还有时间和体力？│────┘
       │    └──────┬───────┘  是
       │           │ 否
       ▼           ▼
    ┌─────────────────────┐
    │   夜间结算            │ ← 消耗食物/恢复体力/事件触发
    │   NPC世界同步运转      │
    └──────────┬──────────┘
               ▼
        ┌──────────────┐
        │ 新的一天       │────▶ 回到顶部
        └──────────────┘
```

**微观循环**（单次行动）：选择 → 执行 → 反馈 → 状态更新
**中观循环**（单日）：起床 → 规划 → 行动 → 夜宿 → 结算
**宏观循环**（一生）：少年学艺 → 青年闯荡 → 壮年成家 → 老年传承 → 死亡/遗产

## 关键设计决策速查

| 维度 | 决策 |
|------|------|
| 时代背景 | 宋朝都城（汴京风格）+ 周边山林农村 |
| 展现形式 | 网页文字冒险，选项以按钮展示 |
| 核心玩法 | 混合型：生存经营 + 角色扮演社交 + 沙盒探索 |
| 时间系统 | 操作驱动回合制，每次操作推进世界，不操作则冻结 |
| 世界运转 | 全世界同步推进，通过7种信息渠道感知远方（详见10） |
| 战斗方式 | 选项驱动，文字描述+按钮决策 |
| 存档机制 | 自动存档 + 手动存档 |
| 死亡机制 | 永久死亡，角色死亡即重来 |
| NPC驱动 | 三层AI：L0 GOAP(≤10) + L1 行为树(≤500) + L2 统计模拟(≤10000) |
| 实体规模 | 同时存在 ≥ 10,000 个可交互对象（NPC/动物/植物/矿物/建筑） |
| 架构模式 | CS架构：服务器计算（ECS+AI） + 前端渲染，WebSocket通信 |
| 多人模式 | 初期单人，架构预留多人扩展能力 |
| 语言风格 | 白话文为主，点缀古文，易读有古意 |
| 游玩周期 | 长局制，单局10小时以上 |
| 技术栈 | TypeScript全栈：Node.js服务端 + 浏览器客户端，WebSocket通信 |
