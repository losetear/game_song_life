# 06 — 演出库（Scene Library）

> 参考《漫野奇谭》(Wildermyth) 的"戏剧图书馆"架构。每个演出都是独立的"剧本"，有严格的演员要求、前置条件、成功/失败分支。

## 设计原则

1. **严格前置条件**：每个演出有明确的发起者条件(actor)和目标条件(target)。如果附近没有符合条件的目标NPC，演出不触发。
2. **成功/失败分支**：互动型演出有对抗判定，成功和失败分别有不同的叙事和效果作用于双方。
3. **角色对称性**：互动对双方都有影响（偷窃→小偷得钱/受害者失钱；偷窃失败→小偷被抓/受害者安全）。
4. **匹配宽容度**：性格条件满足其一即可，关键条件（关系类型、职业、铜钱）必须精确匹配。

## 结果判定类型

| 类型 | 说明 | 判定方式 |
|------|------|---------|
| `certain` | 确定成功 | 只要条件满足就成功（买炊饼、散步） |
| `contested` | 对抗判定 | actor属性 vs target属性（偷窃、打架、说服） |
| `chance` | 概率判定 | 固定概率（采药30%、钓鱼20%） |

## 格式说明

每个演出用结构化块表示：

```
### ID: 演出名
- 目标类别: xxx
- 判定类型: certain/contested/chance
- 权重: N (1-10)

#### 发起者条件 (Actor)
- 性格: [要求] / 禁止: [禁止]
- 职业: [要求] (可选)
- 铜钱: N~M (可选)
- 情绪: [要求] (可选)
- 健康: ≥N (可选)

#### 目标条件 (Target)
- 需要目标: 是/否
- 性格: [要求] (可选)
- 职业: [要求] (可选)
- 铜钱: N~M (可选)
- 关系: [要求] (可选)
- 健康: ≥N (可选)

#### 环境条件
- 地点: [要求] (可选)
- 时段: [要求] (可选)
- 天气: [要求] (可选)
- 季节: [要求] (可选)

#### ✅ 成功
- 叙事: "..."
- 发起者效果: hunger+25, mood+3, copper-5
- 目标效果: (无)
- 关系变化: (无)

#### ❌ 失败 (仅 contested/chance)
- 叙事: "..."
- 发起者效果: mood-5, safety-10
- 目标效果: (无)
- 关系变化: -5
```
