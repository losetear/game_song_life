# 14 - 移动与交互反馈修复

## 问题描述

当前系统存在三个核心缺陷：

### 1. 移动系统不完整

**现状：** 移动选项是硬编码在 SceneTemplate 中的固定按钮（如「去东市」「去码头」），不是根据地图动态生成的。建筑只能进入布庄和药铺，其他建筑（民居、茶楼内部）无法进入。

**目标：** 
- 所有相邻地点都可以移动过去
- 所有建筑都可以进入，进入后看到建筑内的实体
- 移动选项从地图连接关系动态生成，不硬编码
- 建筑内部是一个独立 grid，能看到里面的 NPC 和物品

### 2. 周围人物显示不真实

**现状：** `GET /api/world/nearby` 从 `worldMap.getEntitiesInGrid()` 获取，这是正确的 ECS 查询。但问题是：
- 大多数 NPC 分布在固定 grid 中，玩家移动后可能看不到任何人
- 场景模板中的「NPC对话」是硬编码的模板文字，不是来自真实 NPC
- 前端「周围对象」面板可能没有正确调用 nearby API

**目标：**
- 前端必须从 `GET /api/world/nearby` 加载真实实体
- 场景描述中的 NPC 对话应来自真实的 L0 NPC（如果附近有的话）
- 没有 NPC 时不应显示 NPC 对话

### 3. 交互行为没有产生反馈

**现状：** `executeEntityAction` 虽然写了丰富的文字反馈，但：
- **AP 没有扣除** — `executePlayerAction` 检查了 AP，但 `executeEntityAction` 是在 `executeAction` 的 switch-case default 中调用的，而 AP 扣除逻辑可能没覆盖到这里
- **状态变更不完整** — 某些行为（如偷窃）修改了铜钱，但其他行为的状态变更可能丢失
- **涌现规则的 execute 没被调用** — `executeEntityAction` 走的是自己的 switch-case，没调用 `executeEmergentAction`
- **end-turn 没有重置 AP** — 结束回合后 AP 应恢复到上限

**目标：**
- 所有行为正确扣除 AP
- 涌现规则的 execute 被正确调用，产生状态变更
- end-turn 正确重置 AP
- 前端在行为执行后刷新 AP 显示和周围实体

#### 1.2 动态移动选项生成

新增 `getMovementOptions(playerId)` 方法，根据当前位置的连接 + 附近建筑生成移动选项：

```typescript
getMovementOptions(playerId: number): MovementOption[] {
  const pos = this.em.getComponent(playerId, 'Position');
  const currentGrid = pos?.gridId || 'center_street';
  
  const options: MovementOption[] = [];
  
  // 1. 相邻地点（从 MAP_CONNECTIONS）
  const connections = MAP_CONNECTIONS[currentGrid] || [];
  for (const targetGrid of connections) {
    options.push({
      id: `go_${targetGrid}`,
      name: this.getGridDisplayName(targetGrid),
      icon: this.getGridIcon(targetGrid),
      targetGrid,
      type: 'move',
    });
  }
  
  // 2. 当前位置的可进入建筑（从真实实体）
  const entities = this.worldMap.getEntitiesInGrid(currentGrid);
  for (const eid of entities) {
    if (this.em.getType(eid) === 'building') {
      const building = this.em.getComponent(eid, 'Building');
      const identity = this.em.getComponent(eid, 'Identity');
      if (building) {
        options.push({
          id: `enter_building_${eid}`,
          name: `进入${identity?.name || '建筑'}`,
          icon: '🚪',
          targetGrid: `${currentGrid}_building_${eid}`,
          type: 'enter',
        });
      }
    }
  }
  
  // 3. 如果在建筑内部，显示「离开」选项
  if (currentGrid.includes('_interior') || currentGrid.includes('_building_')) {
    options.push({
      id: 'leave_building',
      name: '离开',
      icon: '🚶',
      targetGrid: this.getParentGrid(currentGrid),
      type: 'leave',
    });
  }
  
  return options;
}
```

### 二、修复周围实体显示

#### 2.1 后端已正确

`getNearbyEntities` 从 `worldMap.getEntitiesInGrid(gridId)` 获取，这是正确的。

#### 2.2 确保建筑内有实体

当玩家进入建筑内部 grid 时，需要确保该 grid 下有实体（NPC、物品等）。方案：
- 进入建筑时，将建筑内的 NPC/物品添加到新 grid
- 或者：nearby API 同时查询当前 grid + 相邻 grid 的实体

#### 2.3 前端必须调用 nearby API

- 页面加载时调用 `GET /api/world/nearby` 并渲染
- 每次执行行为/移动/结束回合后重新调用
- 不使用硬编码的场景描述中的 NPC

### 三、修复交互反馈

#### 3.1 AP 扣除

在 `executePlayerAction` 中，确保 **在执行行为之前扣除 AP**：

```typescript
executePlayerAction(playerId, actionId, params) {
  // 获取 AP 消耗
  const apCost = this.getActionAPCost(actionId);
  
  // 检查 AP
  const ap = this.getPlayerAP(playerId);
  if (ap.current < apCost) {
    return { success: false, message: '行动点不足' };
  }
  
  // 扣除 AP（在执行行为之前！）
  this.setPlayerAP(playerId, ap.current - apCost);
  
  // 执行行为
  const message = this.executeAction(playerId, actionId, params);
  
  // ... 后续回合模拟等
}
```

#### 3.2 涌现行为的执行

在 `executeAction` 的 default 分支中，先尝试调用涌现规则的 execute：

```typescript
default: {
  // 1. 先尝试涌现规则
  const ctx = this.buildInteractionContext(playerId, params?.targetId);
  const feedback = executeEmergentAction(actionId, ctx);
  if (feedback) {
    // 应用反馈到世界状态
    this.applyActionFeedback(playerId, params?.targetId, feedback);
    return feedback.message;
  }
  
  // 2. 再尝试旧的行为系统
  if (params?.targetId) {
    return this.executeEntityAction(playerId, actionId, params, vital, wallet, pos);
  }
  
  return `执行了 ${actionId}`;
}
```

#### 3.3 applyActionFeedback

将涌现规则的反馈应用到世界状态：

```typescript
applyActionFeedback(playerId: number, targetId: number, feedback: ActionFeedback) {
  const playerVital = this.em.getComponent(playerId, 'Vital');
  const playerWallet = this.em.getComponent(playerId, 'Wallet');
  const playerInventory = this.em.getComponent(playerId, 'Inventory');
  
  // 玩家状态变更
  if (feedback.moodChange && playerVital) {
    playerVital.mood = Math.max(0, Math.min(100, playerVital.mood + feedback.moodChange));
  }
  if (feedback.healthChange && playerVital) {
    playerVital.health = Math.max(0, Math.min(100, playerVital.health + feedback.healthChange));
  }
  if (feedback.fatigueChange && playerVital) {
    playerVital.fatigue = Math.max(0, Math.min(100, playerVital.fatigue + feedback.fatigueChange));
  }
  if (feedback.copperChange && playerWallet) {
    playerWallet.copper = Math.max(0, playerWallet.copper + feedback.copperChange);
  }
  if (feedback.itemsGained && playerInventory) {
    playerInventory.items.push(...feedback.itemsGained);
  }
  if (feedback.itemsLost && playerInventory) {
    // 移除对应物品
  }
  
  // NPC 好感度变更
  if (feedback.impressionChange && targetId) {
    const targetMemory = this.em.getComponent(targetId, 'Memory');
    if (targetMemory) {
      targetMemory.impressions[playerId] = (targetMemory.impressions[playerId] || 0) + feedback.impressionChange;
    }
  }
  
  // 记录事件
  this.logEvent('player', '交互', feedback.message.substring(0, 50));
}
```

### 四、前端修改

1. **移动面板**：替换硬编码的场景选项为动态加载的 `getMovementOptions`
2. **周围对象**：每次场景变化后调用 `GET /api/world/nearby` 并渲染
3. **行为执行**：执行后更新 AP 显示 + 刷新 nearby
4. **AP 显示**：从 playerState.ap 实时显示
