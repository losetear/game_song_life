# 16 - 建筑系统深度拓展

## 需求概述

1. 显示当前所在位置和建筑名
2. 进入建筑后切换周围对象，显示建筑内真实物品和人物
3. 去向显示可去的建筑
4. 所有建筑都有内部房间构造、物件陈列、人物关系（谁的房子）
5. 世界监控点击建筑可查看信息

## 一、建筑数据模型扩展

### 1.1 BuildingComponent 扩展

```typescript
export interface BuildingComponent {
  type: string;           // house/shop/teahouse/warehouse/clinic/tavern/temple/government
  name: string;           // 建筑名字（不再靠 ID 取模生成）
  ownerId: number;        // 所有者 NPC ID（0=无主/公产）
  openHours: string;      // 营业时间
  quality: number;        // 建筑品质 1-100
  // 新增
  rooms: Room[];          // 内部房间列表
  description: string;    // 建筑外观描述
}

export interface Room {
  id: string;             // 房间ID（如 "main_hall", "backyard", "kitchen"）
  name: string;           // 房间名（如 "正厅", "后院", "灶房"）
  icon: string;           // emoji
  description: string;    // 房间描述
  objects: RoomObject[];  // 房间内物件
  capacity: number;       // 可容纳人数
}

export interface RoomObject {
  name: string;           // 物件名（如 "八仙桌", "药柜", "织布机"）
  icon: string;           // emoji
  description: string;    // 描述
  interactable: boolean;  // 是否可交互
  itemType?: string;      // 关联物品类型
}
```

### 1.3 NPC-建筑关系

每个 NPC 在 Identity 组件中增加住所字段：

```typescript
export interface IdentityComponent {
  name: string;
  profession: string;
  age: number;
  personality: string[];
  // 新增
  homeId?: number;        // 住所建筑 ID
  workplaceId?: number;   // 工作地点建筑 ID
}
```

在初始化时分配：
- 500 个建筑 → 前 800 个 NPC 各分配一个住所
- 特殊建筑（shop/teahouse/clinic）分配给对应职业 NPC 作为工作地点

### 1.4 建筑初始化重写

将 500 个建筑的生成改为：
- 5% 特殊建筑（shop/teahouse/clinic/tavern）分布在 CITY_GRIDS
- 95% 民居，分布在所有 GRID
- 每个建筑从模板生成房间和物件
- 每个建筑分配 owner（NPC ID）

## 二、API 变更

### 2.1 修改 GET /api/world/nearby

返回建筑时包含更多字段：

```json
{
  "id": 1115,
  "type": "building",
  "name": "孙家旧宅",
  "icon": "🏠",
  "buildingType": "house",
  "ownerName": "孙小花",
  "rooms": [
    { "id": "main_room", "name": "正房", "icon": "🏠" },
    { "id": "kitchen", "name": "灶房", "icon": "🔥" },
    { "id": "courtyard", "name": "院子", "icon": "🌳" }
  ],
  "openHours": "辰-酉",
  "quality": 72,
  "description": "一座青砖灰瓦的老宅，门楣上刻着孙字。"
}
```

## 三、前端变更

### 3.1 位置显示增强

在状态栏显示当前 grid 名称，如果在建筑内则显示建筑名+房间名：

```
📍 汴京 · 中心大街        （室外）
📍 孙家旧宅 · 正房         （室内）
```

### 3.2 进入建筑后的界面

1. 周围对象切换为建筑内的实体（NPC + 物品）
2. 显示房间列表（可点击切换房间）
3. 每个房间显示房间物件
4. 去向变为「离开」+ 其他房间

### 3.3 去向面板

显示格式：
```
【去向】(展开)
  🏠 孙家旧宅 → 进入
  🏠 李家院 → 进入  
  🚶 东市 → 移动
  🚶 码头 → 移动
```

### 3.4 监控面板建筑点击

在监控的搜索/概览中点击建筑 → 弹出建筑详情卡：
- 建筑名+类型+品质
- 所有者
- 房间列表+物件
- 当前在内的人员

## 四、开发优先级

1. **P0**: BuildingComponent 扩展 + 建筑模板 + 初始化重写
2. **P0**: NPC-建筑关系（homeId/workplaceId）
3. **P0**: nearby 返回建筑详情（ownerName/rooms/description）
4. **P0**: 进入建筑后显示真实内部实体
5. **P1**: 建筑 interior API + 房间切换
6. **P1**: 去向面板显示建筑
7. **P1**: 监控面板建筑详情
8. **P2**: 房间物件交互
