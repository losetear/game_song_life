# 13 — 世界监控全景系统

> 核心目标：监控面板是开发者和玩家理解世界的窗口。世界里发生的每一件事、每一个NPC的完整经历，都必须可追溯、可查询。

## 一、设计原则

### 1.1 全透明

世界模拟中发生的所有事情都应该可以查看，没有"黑箱"：
- 每个NPC从诞生到现在的完整行为历史
- 每个回合产生的所有因果事件链
- 经济系统的每次价格变动及原因
- 天气系统的完整变化记录
- 信息传播的完整路径

### 1.2 可追溯

任何一条消息/事件都能反向追溯原因：
- "粮价+15%" → 因为"南郊农田减产" → 因为"连续3天暴雨" → 因为"天气系统推进到雨季"
- "张三偷窃被抓" → 因为"张三饥饿且没钱" → 因为"连续打工没赚到钱" → 因为"码头没有船到港"

### 1.3 实时与历史

- 实时：当前回合的快照（所有实体当前状态）
- 历史：过去每个回合的变化记录（可回放）

## 三、API 设计

### 3.1 现有 API 增强

| API | 当前 | 增强 |
|-----|------|------|
| `GET /api/world/stats` | 基本统计 | +天气信息 +回合数 +经济摘要 |
| `GET /api/world/entities` | 实体列表 | +支持状态筛选 +关系数据 +最近行为 |
| `GET /api/world/entity/:id` | 组件数据 | +完整经历历史 +关系图谱 +记忆内容 |
| `GET /api/world/events` | 事件列表 | +因果链 +回合筛选 +分类筛选 |

### 3.2 新增 API

| API | 说明 |
|-----|------|
| `GET /api/world/events/:tick` | 指定回合的所有事件 |
| `GET /api/world/economy` | 经济全景（价格+供需+走势） |
| `GET /api/world/ecology` | 生态全景（区域统计+趋势） |
| `GET /api/world/weather/history` | 天气历史（最近N回合） |
| `GET /api/world/propagation` | 信息传播链（活跃消息+历史） |
| `GET /api/world/npc/:id/history` | NPC完整行为历史 |
| `GET /api/world/npc/:id/relations` | NPC关系图谱 |

### 3.3 NPC 经历存储

每个 L0 NPC 维护一个行为历史数组：

```typescript
interface NPCHistory {
  tick: number;
  shichen: string;
  day: number;
  action: string;         // "偷窃"
  description: string;    // "在暗巷偷了路人40文"
  cause: string;          // "饥饿(28)+铜钱(3)"
  effects: {              // 行动效果
    hungerChange: number;
    fatigueChange: number;
    copperChange: number;
    movedTo?: string;
  };
  result: string;         // "成功" / "失败-被发现" / "失败-被抓"
}
```

L1 NPC 只保留最近3回合历史（节省内存），L2 无历史。

## 四、实现路线

### Phase 1: NPC 经历系统
- L0 NPC 维护完整行为历史（每回合记录）
- L1 NPC 维护最近3回合摘要
- API: `GET /api/world/npc/:id/history`

### Phase 2: 事件因果增强
- 每条事件增加 cause（触发原因）和 effects（连锁效果）
- API: `GET /api/world/events/:tick`

### Phase 3: 新增监控 API
- 经济全景 /api/world/economy
- 生态全景 /api/world/ecology
- 天气历史 /api/world/weather/history
- 传播链 /api/world/propagation

### Phase 4: 前端监控面板重构
- 替换现有"世界监控"Tab
- 8个子面板（概览/回合事件/NPC档案/经济/生态/天气历史/传播链/搜索）
- 所有数据从 API 实时获取
- 移动端适配（滑动切换面板）

## 五、数据保留策略

| 数据类型 | 保留策略 |
|---------|---------|
| L0 NPC经历 | 全量保留（最多500回合） |
| L1 NPC经历 | 最近3回合 |
| 世界事件 | 最近200条（环形缓冲区） |
| 天气历史 | 最近50回合 |
| 价格走势 | 最近30回合 |
| 传播链 | 活跃消息 + 最近20条已过期消息 |
