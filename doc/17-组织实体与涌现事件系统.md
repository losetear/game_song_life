# 17 - 组织实体与涌现事件系统

## 一、设计目标

当前事件是硬编码模板随机选取。改为：**所有事件由世界中的实体状态变化涌现**。

### 2.2 组织实体组件

```typescript
// FactionComponent — 势力/组织通用组件
export interface FactionComponent {
  name: string;              // 组织名称
  type: 'government' | 'military' | 'merchant' | 'religion' | 'underground' | 'scholar';
  influence: number;         // 影响力 0-100
  treasury: number;          // 金库（文）
  members: number[];         // 成员NPC ID列表
  leaderId: number;          // 首领NPC ID
  territory: string[];       // 控制的gridId列表
  relations: Record<number, number>; // 与其他组织的关系 (-100~100)
  goals: string[];           // 当前目标
  mood: number;              // 组织士气 0-100
}
```

## 三、涌现事件系统重写

### 3.1 核心原则

**事件不再从模板池随机选，而是从世界实体的状态变化涌现。**

```
实体状态变化 → 检查触发条件 → 生成事件 → 写入日志 → 传播
```

### 3.2 事件涌现规则引擎

```typescript
interface EventRule {
  id: string;
  name: string;
  type: WorldEvent['type'];
  // 触发条件
  condition: (world: WorldState) => boolean;
  // 生成事件
  generate: (world: WorldState) => WorldEvent;
  // 优先级（越高越先检查）
  priority: number;
}

interface WorldState {
  tick: number;
  day: number;
  season: string;
  weather: { weather: string; prevWeather: string; description: string };
  factions: FactionComponent[];
  economy: { prices: Record<string, number>; basePrices: Record<string, number> };
  ecology: { regions: any[] };
  npcSummary: { totalActions: number; avgMood: number; avgHunger: number };
}
```

### 3.3 事件规则列表

#### 国际事件（由边境状态涌现）

| 规则ID | 触发条件 | 事件 |
|--------|---------|------|
| intl_trade_route | season==='秋' && 码头帮influence>30 | 丝绸之路商队到达 |
| intl_tributary | day%30===0 && 大相国寺influence>50 | 外国使团朝贡 |
| intl_border | 禁军treasury<20000 | 边境军事紧张 |
| intl_silk_road | 市舶司influence>60 && season!=='冬' | 海外贸易繁荣 |
| intl_border_conflict | 禁军mood<30 || 枢密院influence>90 | 边境冲突消息 |
| intl_peace_treaty | 枢密院mood>70 && 禁军mood>60 | 和平协议 |
| intl_famine_relief | 户部treasury>50000 && 生态欠收 | 赈灾物资调配 |

#### 国家事件（由政府组织涌现）

| nat_infrastructure | 户部treasury>60000 && random>0.5 | 修缮工程 |
| nat_anticorruption | 开封府mood<30 | 御史弹劾 |

#### 地区事件（由区域实体涌现）

| 规则ID | 触发条件 | 事件 |
|--------|---------|------|
| reg_market | 东市商会influence>50 | 东市行情 |
| reg_dock | 码头帮members>3 && 市舶司influence>30 | 码头动态 |
| reg_temple | 大相国寺influence>50 && day%7===0 | 庙会 |

#### 经济事件（由供需涌现）

| 规则ID | 触发条件 | 事件 |
|--------|---------|------|
| eco_price_surge | 任一商品价格>base*1.3 | 价格暴涨 |
| eco_price_drop | 任一商品价格<base*0.7 | 价格暴跌 |
| eco_guild_policy | 布业行会/药材行会 influence变化 | 行会政策 |
| eco_smuggling | 码头帮influence>40 && 市舶司influence<50 | 走私 |

#### 生态事件（由区域统计涌现）

| 规则ID | 触发条件 | 事件 |
|--------|---------|------|
| eco_harvest | 季节+天气+农田统计 | 收成报告 |
| eco_weather_effect | 天气变化 | 天气影响描述 |
| eco_animal | 山林猎物统计 | 动物动态 |

#### 天气事件（由天气状态涌现）

直接从天气系统状态生成，不再随机。

### 3.4 事件生成流程

```
simulateTurn():
  1. 推进时间/天气/经济/生态
  2. 收集世界状态快照
  3. 遍历所有事件规则，检查条件
  4. 条件满足 → 生成事件，写入eventLog
  5. 确保每天至少有以下事件：
     - 天气 1 条（从天气系统读取）
     - 经济 1 条（从价格变化读取）
     - 生态 1 条（从区域统计读取）
     - 国际 1 条（从边境/贸易状态读取）
     - 国家 1 条（从政府组织状态读取）
     - 地区 1 条（从区域实体读取）
     - NPC 行动 3-5 条（从L0 NPC状态读取）
```
