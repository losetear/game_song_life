# 05 — NPC 与 AI 系统

## 核心理念

世界由上万实体组成，NPC有自己的日子要过。但10000个实体不可能都跑完整AI — 需要**分层架构**，让离玩家近的NPC有深度，远处的NPC用简化模拟。

> **架构细节详见 [08-技术架构.md](08-技术架构.md) 的"分层AI系统"章节。本文档侧重AI的行为设计。**

## 分层AI概览

| 层级 | 技术 | 适用对象 | 数量上限 | 更新频率 |
|------|------|---------|---------|---------|
| L0 | GOAP | 玩家视野内的核心NPC | ≤10 | 每次玩家操作 |
| L1 | 行为树 | 同区域其他NPC + 家畜 | ≤500 | 每时辰 |
| L2 | 统计模拟 | 远处所有实体 | ≤10000 | 每天 |

## L0 — GOAP（核心NPC深度AI）

仅用于玩家视野内最重要的NPC（≤10个），保证涌现质量。

### NPC 状态（World State per NPC）

| 状态维度 | 示例值 | 说明 |
|---------|--------|------|
| 位置 | 夜市、家中、农田… | NPC在哪里 |
| 饥饿度 | 0-100 | 需要吃饭 |
| 疲劳度 | 0-100 | 需要休息 |
| 金钱 | 数值 | 可用于交易 |
| 持有物品 | 列表 | 随身携带物 |
| 心情 | 数值 | 影响行为选择 |
| 好感度表 | {npcId: 数值} | 对其他人的态度 |
| 职业 | 身份 | 影响可用行动 |
| 当前目标 | 目标栈 | 当前最优先要做什么 |

### NPC 目标体系

**生存目标（最高优先级）**
- 饥饿了 → 找食物
- 累了 → 回家休息
- 生病了 → 看大夫
- 被攻击 → 逃跑/反击

**日常目标（按职业和性格分配）**
- 商贩：进货 → 摆摊 → 卖货 → 收摊
- 农夫：耕种 → 浇水 → 收割 → 赶集买卖
- 衙役：巡逻 → 检查 → 抓贼 → 交接班
- 书生：读书 → 作文 → 偶尔社交
- 大夫：坐堂 → 看诊 → 采药
- 小偷：踩点 → 找目标 → 偷窃 → 销赃 → 躲藏
- 厨子：备料 → 做菜 → 卖食 → 收摊

**长期目标（影响人生轨迹）**
- 攒钱买房
- 科举中举
- 找对象结婚
- 报仇雪恨
- 成为行业翘楚

### NPC 性格模型

每个NPC有性格标签组合，影响决策权重：

| 性格标签 | 特点 | 行为倾向 |
|---------|------|---------|
| 勤劳 | 优先工作 | 工作相关行动权重+50% |
| 懒惰 | 拖延摸鱼 | 休息娱乐权重+50% |
| 贪婪 | 追求利益 | 金钱相关行动权重+50% |
| 善良 | 帮助他人 | 利他行为权重+50% |
| 狡猾 | 精于算计 | 欺骗类选项开放 |
| 暴躁 | 容易发怒 | 冲突概率增加 |
| 胆小 | 回避危险 | 逃跑概率增加 |
| 好客 | 乐于交友 | 主动社交概率增加 |

> 一个NPC通常有2-3个性格标签组合，如"勤劳+善良"或"狡猾+贪婪"。

### NPC 记忆系统（原稿缺失，补充）

每个NPC维护一份记忆日志，影响其行为决策：

| 记忆类型 | 保存时间 | 示例 | 影响 |
|---------|---------|------|------|
| 近期事件 | 3-7天 | "三天前张三偷了我的钱" | 直接影响对某人态度 |
| 长期记忆 | 永久 | "去年冬天那场大雪冻死很多人" | 偶尔在对话中提及 |
| 人物印象 | 永久（缓慢衰减） | "那个卖炊饼的伙计人不错" | 影响好感度变化速度 |
| 交易记录 | 30天 | "上周从东市进了一批布，花了两贯" | 影响定价和进货决策 |
| 恩怨记录 | 永久 | "这人救过我的命" / "这人害我坐牢" | 特殊交互选项解锁 |

**记忆衰减规则**：
- 近期事件随时间转为长期记忆（细节模糊）
- 对未再接触的人，好感度每30天自然衰减5点
- 恩怨记录不会衰减，但严重程度可随时间略微降低

### GOAP 行动表（原稿缺失，补充）

每个行动需要定义：前置条件、执行效果、消耗代价

```
行动示例：

【买炊饼】
  前置：位置=街边/夜市 AND 金钱>=10 AND 饥饿度<100
  效果：饥饿度+30, 金钱-10, 持有物品+炊饼x1
  消耗：时间-0.3时辰
  可用者：所有人

【偷窃】
  前置：位置=暗处/拥挤处 AND 性格包含"狡猾" OR 职业=小偷
  效果：成功(60%)→金钱+目标身上钱*0.5, 失败(40%)→被通缉
  消耗：时间-1时辰
  可用者：小偷、狡猾角色

【坐堂看病】
  前置：职业=大夫 AND 位置=药铺 AND 时辰=巳-未
  效果：金钱+30~100, 声望+1
  消耗：时间-2时辰
  可用者：大夫

【巡逻】
  前置：职业=衙役 AND 时辰=辰-戌 AND 位置=城内
  效果：发现犯罪概率+50%, 声望+1
  消耗：时间-2时辰, 疲劳+20
  可用者：衙役
```

> **设计要点**：行动表初期控制在 30-50 个行动以内，后续按职业逐步扩展。每个行动都需要明确的数值定义，避免模糊描述。

## L1 — 行为树（同城NPC简化AI）

用于玩家同区域但不在视野内的NPC（≤500个）。牺牲涌现深度换取性能。

### 行为树模板（按职业）

```
[通用NPC行为树骨架]
  Root → Selector
  ├── 满足生存需求？→ 执行生存行动
  ├── 工作时间？     → 执行职业行动
  ├── 社交时间？     → 执行社交行动（随机概率）
  └── 默认          → 待在原地/回家

[商贩模板]
  工作行动 = Selector
    ├── 库存充足 → 卖货
    ├── 库存不足 → 进货 → 卖货
    └── Random(15%) → 去茶馆

[农夫模板]
  工作行动 = Selector
    ├── 春季 → 播种
    ├── 夏季 → 浇水/除草
    ├── 秋季 → 收割
    └── 冬季 → 休息/赶集

[衙役模板]
  工作行动 = Selector
    ├── 发现犯罪 → 追捕
    └── 正常 → 巡逻（按路线点移动）

[小偷模板]
  工作行动 = Selector
    ├── 有目标且无人看 → 偷窃
    ├── 无目标 → 踩点（在人群中游荡）
    └── Random(20%) → 去销赃
```

### L1 NPC不维护的项目（与L0的区别）

| 移除项 | 原因 | 替代方案 |
|--------|------|---------|
| 完整记忆系统 | 500个NPC各持记忆太贵 | 仅保留3条近期事件 |
| GOAP规划器 | 太慢 | 行为树（毫秒级） |
| 动态对话生成 | 无需对话 | 预设对话模板 |
| NPC间好感度 | 太多关系对 | 仅保留与玩家和L0 NPC的关系 |

### L1 NPC状态更新

```
每时辰（非每动作）批量更新：
  1. 按职业分组
  2. 每组批量计算行为树
  3. 批量更新位置和状态
  4. 如有重要结果（犯罪被抓等），推送至事件队列

预计性能：500个NPC × 行为树 ~ 0.01ms/个 = 5ms/时辰
```

## L2 — 统计模拟（远处实体）

城外和远处的实体不模拟个体行为，只做区域级统计。

### L2 统计规则

```
[区域: 城外农田]
  每天计算一次：
    区域内农夫数 = 从L1/L2人口统计获取
    区域面积 = 固定值
    天气修正 = 从WeatherSystem获取
    季节修正 = 查表

    产量 = 面积 × 基础亩产 × 天气修正 × 季节修正 × 劳力修正
    农夫消耗 = 农夫数 × 日消耗
    净产出 = 产量 - 消耗

    粮食供给 → 推入EconomySystem
    如果 净产出 < 0 → 部分农夫可能迁入城中

[区域: 山林]
  每天计算一次：
    野生动物数 = 上日数量 × (1 + 繁殖率 - 狩猎率 - 自然死亡率)
    如果 野生动物数 > 容量上限 → 设为上限（防止无限增长）
    如果 野生动物数 < 最小值 → 触发"此山野兽稀少"消息

    可采集资源 = 基于季节和天气计算
    采集后按规则重生（不同资源不同重生时间）

[区域: 河流]
  每天计算一次：
    鱼获量 = 基于季节 + 捕鱼人数
    水匪活动 = 基于治安值随机
```

### L2 → L0/L1 的升级触发

当玩家移动到某个区域时，该区域的L2实体需要"具象化"为L0/L1：

```
具象化流程：
  1. 读取区域统计数据（如"山林有15只兔子"）
  2. 按规则在Grid中放置具体实体（15只兔子分布在合适位置）
  3. 为每个实体创建ECS组件（Vital, Growth等）
  4. 核心NPC升级为L0 GOAP
  5. 其他NPC/动物升级为L1 行为树

离开时反序列化：
  1. 统计L0/L1实体的结果（兔子现在剩12只了）
  2. 销毁ECS组件，释放内存
  3. 将统计数据写回L2
```

> **关键**：具象化必须确定性（基于种子），这样玩家反复进出同一区域，实体分布一致。

### NPC 日程（基于GOAP动态生成）

NPC不按固定脚本走，但会有习惯倾向：

```
[商贩王老板 — 勤劳+贪婪]
  卯时 → 起床，去东市进货
  辰时 → 回铺子开张
  巳-午 → 卖货，与客人交谈
  未时 → 午饭，算账
  申-酉 → 继续营业
  戌时 → 关门，去酒楼社交
  亥时 → 回家睡觉

  * 特殊：如果听说哪有好货，可能临时去进货
  * 特殊：如果被人欠钱，可能去找人讨债
  * 特殊：如果听说夜市热闹，可能晚关门去摆摊
```

## 涌现事件示例

NPC的独立决策会导致意想不到的事件涌现：

### 示例1：小偷被抓
```
1. 张三（混混）目标：偷钱
2. 张三在夜市踩点，看到喝醉的客商
3. 张三偷了客商的钱袋
4. 钱捕头刚好巡逻经过
5. 客商发现丢钱大喊
6. 钱捕头追捕张三
7. 张三被抓入狱

→ 玩家可能目击全过程
→ 玩家可能帮助追贼（提升声望）
→ 玩家可能在张三被抓前被张三栽赃
```

### 示例2：粮价飞涨
```
1. 夏季暴雨，秋收减产（系统事件）
2. 城外农户收成不好（NPC农夫状态更新）
3. 粮铺进货价格上涨
4. 王掌柜（贪财）提高粮价
5. 市民抱怨，部分人囤粮
6. 官府出面平抑粮价
7. 有人趁乱偷粮

→ 玩家如果有存粮可以高价卖出
→ 玩家如果种田，今年收入减少
→ 玩家可以报官/帮忙维持秩序
```

### 示例3：瘟疫传播
```
1. 某NPC从外地来，携带疾病
2. 接触过的人都可能感染
3. 大夫发现异常，发出警告
4. 感染者增多，城里人心惶惶
5. 官府封城/隔离
6. 药铺药品紧缺，价格飞涨
7. 大夫研制药方

→ 玩家可能被感染（取决于接触史）
→ 玩家可以帮忙采药/送药
→ 玩家可以趁机囤药高价卖（但声望下降）
```

## NPC对话生成规则

### 对话结构
```
[问候/反应] + [内容] + [暗示/需求]

示例：
"这位客官，今儿个来得早啊。"
（王掌柜看到玩家，根据时间/关系/心情生成问候）

"最近城里不太平，您出门可要当心。"
（基于王掌柜知道的消息，善意提醒）

"对了，我那儿有批上好的苏绣，要不要看看？"
（王掌柜当前目标：卖货，所以主动推销）
```

### 对话影响因素
| 因素 | 影响 |
|------|------|
| 好感度 | 高→热情；低→冷淡/拒绝交谈 |
| 时间 | 早上→问候"早"；深夜→奇怪为什么还在 |
| 天气 | 雨天→抱怨天气；晴天→心情好 |
| 近期事件 | 发生大事→谈论大事 |
| NPC当前目标 | 目标相关→主动提及 |
| 玩家声望 | 高声望→恭敬；低声望→轻视 |
