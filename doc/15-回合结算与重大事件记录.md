# 15 - 回合结算与重大事件记录

## 需求

结束回合时，整个世界的所有 AI 层级都要进行完整结算，所有重大事件必须记录并展示给玩家。

## 当前状态

`simulateTurn()` 已覆盖所有层级：
1. 时间推进
2. 天气系统
3. L0 NPC 行动（10个核心 NPC，状态驱动 GOAP）
4. L1 NPC 批量行动（简化行为树）
5. L2 区域统计更新
6. 经济系统更新（供需/天气/区域影响）
7. 生命系统衰减
8. 因果事件引擎
9. 随机事件

**问题：** `turnSummary` 只返回 `events: number` 和 `npcActions: number`，没有具体事件内容。前端无法展示"这一回合发生了什么"。

## 方案

### 一、增强 turnSummary

```typescript
turnSummary: {
  shichen: string;
  day: number;
  tick: number;
  
  // 各层级结算摘要
  l0Actions: { npcName: string; action: string; result: string }[];  // L0 NPC 具体行动
  l1Summary: { total: number; highlights: string[] };                 // L1 批量行动摘要
  
  // 重大事件（必须有）
  majorEvents: {
    type: 'weather' | 'economy' | 'ecology' | 'npc' | 'player';
    title: string;        // 简短标题
    detail: string;       // 详细描述
    impact: 'critical' | 'important' | 'minor';  // 影响等级
  }[];
  
  // 环境变化
  weatherChange: string | null;  // 如 "晴→暴雨"
  priceChanges: Record<string, { change: string; reason: string }>;
  
  // 统计
  totalEvents: number;
  npcActions: number;
}
```

### 二、重大事件判定规则

**critical（必须通知玩家）：**
- 天气突变（晴→暴雨/大雪/冰雹）
- 粮价涨跌 >20%
- L0 NPC 健康降到 0（死亡）
- L0 NPC 遭遇重大变故
- 洪涝/旱灾/瘟疫

**important（应该通知）：**
- 物价涨跌 >10%
- L0 NPC 状态大幅变化（饥饿<20，心情<20）
- 季节更替
- 天气变化（晴→雨/阴→晴）
- 传播链扩散（消息传遍全城）

**minor（可选通知）：**
- NPC 日常行动
- 小幅物价波动
- 天气微变

### 三、L0 NPC 行动记录增强

当前 `simulateL0Action` 只做了简单的状态驱动动作。增强为：
- 每个动作生成具体的描述文字
- 记录动作的原因（如"张三去买饭因为他饿了"）
- 如果动作失败也记录（如"张三想买饭但没钱"）

### 四、前端展示

在 Demo 面板（主游戏区域）中：
1. 执行行为后 / 结束回合后，显示「本回合大事记」面板
2. critical 事件红色边框 + 加粗
3. important 事件金色边框
4. minor 事件普通样式
5. 显示后 3 秒自动淡出或可手动关闭

## API 变更

`POST /api/end-turn` 和 `POST /api/action` 返回的 `turnSummary` 增强为上述结构。

新增字段不需要改路由，只改 `TickResult` 类型和填充逻辑。
