# 19 - 组织成员归属与涌现逻辑

## 目标

1. NPC 实体携带 `factionId`，明确知道自己的组织归属
2. 前端 NPC 详情面板显示组织信息，点击可跳转到组织详情
3. 组织属性参与事件涌现逻辑

## 改动范围

### 后端

#### 1. `src/ecs/types.ts` — IdentityComponent 扩展
```typescript
export interface IdentityComponent {
  name: string;
  profession: string;
  age: number;
  personality: string[];
  factionId?: number;      // 所属组织实体 ID（0 或 undefined = 无组织）
  factionRole?: 'leader' | 'member';  // 组织角色
}
```

#### 2. `src/data/entities.ts` — 初始化时回写 factionId
- `generateFactions()` 中，分配成员后给每个 member NPC 的 Identity 组件写入 `factionId` 和 `factionRole`
- 首领 NPC 设 `factionRole: 'leader'`

#### 3. `src/world/worldEngine.ts` — 新增查询方法
- `getNpcFaction(npcId: number): { id: number; name: string; role: string } | null`
- `getFactionMembersDetailed(factionId: number)`: 返回完整成员信息（含 profession, health, mood, gridId）

#### 4. `src/server/gameServer.ts` — API 增强
- `GET /api/world/entity/:id` 响应中增加 `factionId` 和 `factionName` 字段
- `GET /api/world/factions/:id` 详情中成员信息更丰富（含 gridId, health, mood）

#### 5. `src/world/emergenceRules.ts` — 组织涌现规则（新增 5-8 条）
- **同组织认同**：对同组织 NPC 好感度加成，交易/互动有折扣
- **跨组织冲突**：敌对组织 NPC 互动可能触发冲突事件
- **组织任务**：首领 NPC 可能发布组织相关任务
- **势力范围**：在组织控制区域内，该组织 NPC 出现概率更高
- **组织资源**：组织金库影响成员钱包（俸禄/分红）
- **士气影响**：组织 mood 低时，成员 mood 也受影响

### 前端

#### 6. `client/index.html` — NPC 详情面板
- NPC 详情中显示「所属组织」标签（含名称 + 角色）
- 点击组织名称跳转到组织详情面板
- 组织详情中点击成员跳转到 NPC 详情（已有 `showNPCInfo`）

## 数据流

```
NPC 创建 → generateFactions() 分配成员 → Identity.factionId 回写
                                                    ↓
entity/:id API → 返回 factionId + factionName → 前端展示
                                                    ↓
emergenceRules → 检查 ctx.target.identity.factionId → 触发组织相关涌现
```

## 优先级
P0 — 核心数据完整性需求，组织系统没有双向关联就无法支撑后续玩法
