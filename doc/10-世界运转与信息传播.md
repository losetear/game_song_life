# 10 — 世界运转与信息传播

## 核心规则

**世界不会自动运转。玩家每次操作（包括"原地等待"）触发一次全局世界更新。**

```
玩家操作 → 消耗对应时长 → 全世界同步推进该时长 → 结果反馈给玩家

示例：
  玩家选择"买炊饼"(0.3时辰) → 全世界推进0.3时辰 → 玩家看到附近变化 + 收到远方消息
  玩家选择"原地歇会儿"(1时辰) → 全世界推进1时辰 → 玩家观察到更多变化
  玩家选择"在田埂上坐一个下午"(3时辰) → 全世界推进3时辰 → 大量事件发生
  玩家什么都不选 → 时间静止，世界冻结
```

## 一、单次世界更新流程

玩家每点一个按钮，服务器执行以下完整流程：

```
玩家操作到达服务器
        │
        ▼
┌───────────────────────────────────────┐
│  1. 消耗时间                           │
│     本次操作消耗 T 个时辰               │
│     世界时钟 += T                       │
│     判断是否跨日/跨季节                  │
└───────────────────┬───────────────────┘
                    ▼
┌───────────────────────────────────────┐
│  2. 执行玩家操作                        │
│     更新玩家ECS状态                     │
│     计算操作结果（成功/失败/部分）        │
└───────────────────┬───────────────────┘
                    ▼
┌───────────────────────────────────────┐
│  3. L0 区域更新（玩家视野内 ≤50实体）   │
│     GOAP规划 × 命名NPC                  │
│     完整状态变化                         │
│     完整记忆更新                         │
│     生成本地事件（看到/听到/闻到）        │
└───────────────────┬───────────────────┘
                    ▼
┌───────────────────────────────────────┐
│  4. L1 区域更新（同城 ≤500实体）        │
│     行为树批量执行                       │

│     检测是否产生可传播事件                │
└───────────────────┬───────────────────┘
                    ▼

│     远方消息（通过信息渠道到达）          │
│     状态变化 + 新选项                    │
└───────────────────────────────────────┘
```

## 二、"等待"系列操作

玩家不一定要做事，可以选择原地等待，让世界流逝：

| 等待选项 | 消耗时间 | 触发世界更新 | 适合场景 |
|---------|---------|------------|---------|
| 小憩片刻 | 0.5时辰 | 完整更新 | 等人、歇脚 |
| 歇一个时辰 | 1时辰 | 完整更新 | 等店铺开门、等人 |
| 打个盹 | 2时辰 | 完整更新 | 午后休息 |
| 原地待半天 | 3时辰 | 完整更新 | 等天黑/天亮 |
| 原地待一天 | 到当日结束 | 完整更新+跨日结算 | 快速跳过无聊时段 |
| 连续等待多日 | 1-7天 | 每日完整更新 | 养伤/等消息/过冬 |

> 等待期间玩家的饥饿/疲劳照常变化，不会"白等"。等待只是不做事，不是暂停需求。

### 等待时玩家能观察到什么

```
示例：玩家在茶馆"歇一个时辰"

主场景：
  你要了一壶茶，靠在椅背上歇息。茶馆里人声嘈杂。

等待期间观察到的（世界更新结果）：

  [视觉] 一个穿着官服的人匆匆从街上走过，后面跟着两个衙役。
  [听觉] 隔壁桌的两个人在小声议论："听说城南昨晚遭了贼……"
  [嗅觉] 厨房飘来一股炖羊肉的香味。
  [消息] 刘寡妇边擦桌子边跟你说："哎，你知道不，城外的陈猎户
         昨天在山里打到一头大鹿，今天东市有新鲜鹿肉卖呢！"
  [远方] （如果等待更长时间，可能收到更多远方消息）

  一个时辰过去了。
  你的饥饿度下降了，精神好了些。

  [继续喝茶]  [离开茶馆]  [和刘寡妇多聊聊]  [去看看鹿肉]
```

## 三、L2 叙事引擎（核心新增）

**问题**：L2区域只有统计数字，玩家看不到。
**解决**：L2统计变化时，自动生成叙事事件，通过信息渠道传递给玩家。

### 叙事事件生成规则

```
每种L2统计变化都有对应的叙事模板：

[农田区域 — 丰收]
  触发：区域产量 > 正常值 × 1.5
  叙事："听说今年东郊的稻子长得格外好，稻穗都压弯了腰。"
  经济影响：粮价-10%

[农田区域 — 灾荒]
  触发：区域产量 < 正常值 × 0.3
  叙事："南边的田地遭了蝗灾，颗粒无收，好些农人拖家带口往城里来了。"
  经济影响：粮价+30%，城中多出流民NPC

[山林区域 — 野兽增多]
  触发：野生动物数量 > 容量 × 0.8
  叙事："近来山中野兽出没频繁，已有多户人家的牲畜被叼走。"
  可触发：官府悬赏任务

[山林区域 — 猎物稀少]
  触发：野生动物数量 < 容量 × 0.2
  叙事："猎户们都说最近山里打不到东西，怕是猎多了，得歇歇。"
  经济影响：兽肉/皮毛涨价

[河流区域 — 鱼获丰收]
  触发：当季鱼获量 > 正常 × 1.5
  叙事："码头那边热闹得很，今儿的鱼获比往常多了一倍不止。"
  经济影响：鱼价下降

[河流区域 — 水匪出没]
  触发：水匪活动值 > 阈值
  叙事："听说汴河下游最近不太平，好几条船被劫了，船家都不敢往下游走。"
  可触发：官府征召/镖局任务

[区域间 — 人口流动]
  触发：某区域人口变化 > 10%
  叙事（流入）："最近城里多了不少外地口音的人，说是老家遭了灾。"
  叙事（流出）："好多人都说城里活不下去了，有不少人往南边去了。"

[经济 — 物价异动]
  触发：某商品价格变化 > 20%
  叙事（涨）："最近米价涨了不少，粮铺门口排起了长队。"
  叙事（跌）："听说新到一批货，市面上的布匹便宜了不少。"
```

### 叙事事件池

```
每个区域每天根据统计变化生成 0-3 条叙事事件：

事件重要度：
  ★☆☆ 闲闻琐事（"村口老王家的牛生了个双犊。"）
  ★★☆ 有用消息（"东市新到了一批便宜的棉布。"）
  ★★★ 重要事件（"城外发现流寇踪迹，官府加强了城门检查。"）
  ★★★★ 重大事件（"瘟疫从南方传来了，城里人心惶惶。"）
  ★★★★★ 改变世界（"北方战事吃紧，朝廷征调粮草。"）

只有重要度 ≥ ★★☆ 的事件才会传播到玩家耳中。
低重要度事件存在事件池中，NPC可能会偶尔提及。
```

## 四、信息传播系统

### 传播渠道

```
┌────────────────────────────────────────────────────────┐
│                    信息传播渠道                          │
├────────────────────────────────────────────────────────┤
│                                                        │
│  [直接感知] — 玩家亲眼看到/亲耳听到                      │
│  范围：当前Grid + 相邻Grid                              │
│  延迟：0（即时）                                        │
│  详细度：最高                                           │
│  示例：看到张三偷东西、听到远处喊"抓贼"                   │
│                                                        │
│  [NPC转述] — 与玩家交谈的NPC主动告知                     │
│  范围：NPC所知道的区域（取决于NPC活动范围和社交网络）      │
│  延迟：1-12时辰（取决于消息传播链长度）                   │
│  详细度：中高                                           │
│  示例："哎你知道吗，昨晚码头那边又出事了……"              │
│                                                        │
│  [茶馆/酒楼八卦] — 在社交场所听到                        │
│  范围：全城（茶馆是消息集散地）                           │
│  延迟：半天-1天                                         │
│  详细度：中（可能夸大或遗漏）                            │
│  示例：茶馆里几个人在大声议论粮价上涨                     │
│                                                        │
│  [官方告示] — 官衙门口张贴的布告                         │
│  范围：全城                                             │
│  延迟：事件发生后官府决定张贴的时间                       │
│  详细度：官方版本（可能隐瞒）                            │
│  示例："告示：近日城南有流寇出没，百姓夜间出行须小心"     │
│                                                        │
│  [街巷传闻] — 走在街上偶尔听到的路人闲话                  │
│  范围：城中正在流传的消息                                │
│  延迟：1-3天                                            │
│  详细度：低（传闻经常失真）                              │
│  示例：路过时听到两个妇人在说"听说张寡妇家……"            │
│                                                        │
│  [行商消息] — 从外地来的商人带来的远方消息                │
│  范围：其他城镇/远方                                     │
│  延迟：数天到数周                                       │
│  详细度：低（道听途说）                                 │
│  示例："从南边来的客商说，那边正在闹瘟疫……"              │
│                                                        │
│  [书信] — 通过钱庄/驿站传递的书信                        │
│  范围：特定地点的人                                      │
│  延迟：按距离计算（同城当天，异地数天）                   │
│  详细度：取决于写信人                                   │
│  示例：收到师傅的来信，说他在南方找到了稀有的药材          │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### 传播延迟规则

```
事件发生地 → 玩家感知的延迟：

  同Grid:        即时（直接看到/听到）
  同Area不同Grid: 0.5-2时辰（通过NPC/声音传播）
  同城不同Area:   半天-1天（通过茶馆/社交传播）
  城外附近:       1-2天（通过进城的人带来）
  远方:           3-7天（通过商人/驿站/书信）

  影响因素：
  - 事件重要度：越重要传播越快（重大事件半天全城皆知）
  - 玩家是否在社交场所：在茶馆/酒楼更容易听到消息
  - 玩家声望/人脉：高声望者消息来得更快
  - NPC好感度：高好感度的NPC会主动告诉你消息
```

### 信息失真系统

消息传播过程中可能失真（传话传错了）：

```
原始事件：张三在夜市偷了一个客商的钱袋

传播1次后（2时辰）：
  "听说夜市那边有人被偷了。"(丢失了具体人物信息)

传播3次后（半天）：
  "夜市上有人偷了好几十贯钱！"(金额被夸大)

传播5次后（1天）：
  "听说夜市来了个大盗，一次偷了好几百贯！"(严重失真)

失真规则：
  - 每多传播一次，细节有20%概率丢失
  - 数字有30%概率被夸大/缩小
  - 人物有10%概率被张冠李戴
  - 重要度越高，失真越少（大事件传得准确）
```

每次玩家操作后，服务器返回三类信息：

```

  "scene": "你买了一碗热腾腾的羊肉汤，暖意从胃里升起。",

  // 2. 附近感知（当前Grid/相邻Grid的事件）
  "nearbyEvents": [
    { "sense": "视觉", "text": "对面布庄的王掌柜正在和一个客商激烈讨价还价。" },

    {

      "text": "旁边几人在议论：'城外南边的村子遭了蝗灾，好多人往城里逃呢。'",

```

### 远方消息的推送节奏

不是所有消息都堆给玩家，有节制地推送：

```
每次操作最多推送：
  nearbyEvents:  ≤ 3条
  distantNews:   ≤ 2条（按重要度排序，只推最重要的）

积压的消息怎么办：
  - 存入"消息队列"
  - 下次操作时，旧的低重要度消息可能被更高重要的挤掉
  - 玩家可以主动去茶馆/官衙"打听消息"，一次性获取更多积压消息
  - 玩家可以翻看"消息记录"面板查看历史消息
```

## 六、各渠道的触发时机

不同渠道在不同情况下才会推送消息给玩家：

| 渠道 | 触发条件 | 推送方式 |
|------|---------|---------|
| 直接感知 | 玩家在事件发生地附近 | 场景描述中自然融入 |
| NPC转述 | 玩家与有关NPC交互时 / NPC主动找你 | 对话中出现 |
| 茶馆八卦 | 玩家在茶馆/酒楼时 | 环境描述中提及 |
| 官方告示 | 玩家经过官衙或城门时 | 场景选项中出现"看告示" |
| 街巷传闻 | 玩家在街上走动时 | nearbyEvents中出现 |
| 行商消息 | 码头/集市有外地商人时 | 交易时附带消息 |
| 书信 | 玩家回家或去钱庄时 | 触发"有你的信"事件 |

## 七、跨日/跨季节大事件

当玩家的操作跨越了时间节点，会触发更全面的更新：

### 跨日结算

```
当玩家操作导致世界时钟跨过"亥时末"（一天结束）：

1. 日常结算
   ├── 全体NPC饥饿/疲劳/健康日变化
   ├── L2区域全面统计更新
   ├── 作物生长推进
   └── NPC好感衰减

2. 生成当日事件摘要
   ├── 今日城中大事（2-3条）
   ├── 今日城外动态（1-2条）
   └── 今日物价变动

3. 次日初始化
   ├── 天气确定
   ├── NPC日程重置（基于GOAP/行为树重新规划）
   ├── 店铺库存刷新
   └── 日期推进

推送给玩家：
  "夜深了，你回到住处歇下。"
  "（次日清晨）"
  "今日天气：晴。"
  "昨日城中大事：张三因偷窃被判打二十大板。"
  "城外消息：南郊农田遭了虫害，今年收成怕是不太好。"
  "市价变动：米价上涨了15%。"
```

### 跨季节结算

```
当世界时钟跨过季节分界线：

1. 季节变化
   ├── 新季节的天气概率表切换
   ├── 农作物状态切换（春播/夏长/秋收/冬藏）
   └── 野生动物种群重新计算

2. 大范围经济调整
   ├── 季节性商品价格调整
   ├── 供需关系重算
   └── 区域间贸易流动

3. 季节性事件池刷新
   ├── 春：春耕、花朝节、春闱（科举）
   ├── 夏：端午、酷暑、防洪
   ├── 秋：中秋、秋收、秋闱
   └── 冬：冬至、大雪、年关

推送给玩家：
  "时令已至（春/夏/秋/冬），万物复苏/炎热/萧瑟/沉寂。"
  + 季节性事件通知
  + 物价变化提示
  + 可用活动提示（如"科举将近"）
```

## 八、NPC信息携带机制

NPC在城中移动时，就像行走的"信息载体"：

```
每个NPC维护一个消息队列（memory中的事件列表）：

NPC消息获取：
  - 亲眼目睹的事件（L0感知范围内的）
  - 其他NPC告诉TA的（社交交互时传播）
  - 在茶馆/酒楼听到的（社交场所自动获取）

NPC消息传播：
  - 与其他NPC交谈时，有概率传播自己知道的消息
  - 与玩家交谈时，可能主动提及（取决于好感度+话题相关性）

示例：
  周农夫（城外来的）
  知道：城外南郊遭了虫害、自己今年收成不好
  传播路径：周农夫进城赶集 → 在茶馆和人聊天 → 消息进入茶馆传闻池
  玩家获取：去茶馆就能听到，或者直接和周农夫交谈也能知道
```

### 旅行商人 — 远方消息的特殊载体

```
旅行商人是远方消息的主要来源：
  - 定期从城外/远方来到城中
  - 带来远方区域的L2叙事事件
  - 消息比普通渠道更丰富但可能不太准确

触发方式：
  - 码头偶尔有外地商船靠岸
  - 东市/西市偶尔出现外地客商
  - 客栈偶尔有外地住客

交互：
  [和外地商人攀谈] → 消耗0.5时辰 → 获取远方消息 + 可能触发交易机会
```

### 主动打听机制

```
在社交场所可以主动打听消息：

[茶馆] 选择"听听茶馆里的消息" → 消耗0.5时辰 → 获取3-5条积压消息
[酒楼] 选择"听食客们聊天"    → 消耗0.5时辰 → 获取2-3条消息
[官衙] 选择"看看告示栏"       → 消耗0.2时辰 → 获取官方通告
[码头] 选择"和船家聊天"       → 消耗0.3时辰 → 获取水路/远方消息
[集市] 选择"打听行情"         → 消耗0.5时辰 → 获取物价+供需趋势
```
