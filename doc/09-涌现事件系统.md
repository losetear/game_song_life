# 11 — 涌现事件系统（级联反应 + 动态涌现）

> 核心原则：世界发生的一切都必须有因果链。没有"凭空出现"的事件。所有变化通过**事件**传播，一个事件可触发多个系统的连锁响应。

---

## 一、设计原则

### 1.1 禁止随机事件

**❌ 错误做法：**
- 远方消息从预设数组随机选
- NPC行为从模板数组随机选
- 事件没有因果依据

**✅ 正确做法：**
- 事件必须由世界状态变化触发
- NPC行为必须由其状态（饥饿/疲劳/铜钱）驱动
- 消息必须由真实事件传播而来
- 远方消息必须有L2统计支撑

### 1.2 三层因果架构

```
世界状态变化（L2统计 / L1行为 / L0 GOAP）
    ↓ 触发条件检查
事件生成（基于状态差异，非随机模板）
    ↓ 传播链
叙事描述（基于事件内容，非预设文案）
```

### 1.3 NPC 知识来源规则

NPC 只能谈论他们**实际感知**到的信息：
- 同一 Grid 内发生的事件（亲眼看到/听到）
- 从其他 NPC 传播来的消息（传播链）
- 在茶馆/集市等社交场所获取的（信息集散地）
- **绝对不能**谈论没有任何依据的事情

## 二、级联反应系统

### 2.1 事件类型

| 事件类型 | 说明 | 触发源 |
|----------|------|--------|
| weather_changed | 天气变化 | 天气系统 |
| time_advanced | 时间推进 | 时间系统 |
| season_changed | 季节变化 | 时间系统 |
| entity_created | 实体创建 | 世界管理器 |
| entity_destroyed | 实体销毁 | 多个系统 |
| player_action | 玩家操作 | 交互系统 |
| npc_behavior | NPC行为 | NPC AI |
| npc_interaction | NPC间互动 | NPC AI |
| economy_transaction | 交易 | 经济系统 |
| area_entered | 进入区域 | 移动系统 |
| festival_started | 节日开始 | 时间系统 |
| disaster_triggered | 灾害触发 | 天气系统 |

### 2.2 级联控制参数

| 参数 | 值 | 说明 |
|------|------|------|
| 最大传播深度 | 10层 | 超过自动截断 |
| 每层效果衰减 | ×0.7 | 越远影响越小 |
| 每 Tick 事件预算 | 500个 | 超出留到下个 Tick |
| 去重 | 同事件+同实体同Tick | 只触发一次 |
| 单系统限流 | 100个子事件/Tick | 超过限流该系统 |

### 2.3 级联示例——一场暴雨

```
天气系统发出 "weather_changed" { type: "heavy_rain" }

Tier 0 — 即时反应（< 2秒）
├── 地面：所有户外地块湿度+50，石板路变"湿滑"
├── NPC：户外 NPC 评估避雨
│   ├── 有伞 → 撑伞
│   ├── 无伞 → 跑向最近屋檐
│   └── 商人 → 收摊
├── 动物：鸟归巢、猫躲雨、鱼活跃
├── 物品：露天物品受潮（布匹质量下降、火堆熄灭）
└── 叙述：生成雨景文字描述

Tier 1 — 短期影响（10秒-1分钟游戏时间）
├── 经济：伞需求×3（涨价），户外商品供给×0.5
├── 农业：所有农田水分+30，作物生长加速
├── 地形：低洼处积水，泥土路变泥泞
├── NPC：淋雨的NPC可能感冒 → 去药铺买药
└── 记忆：NPC 记住"那场暴雨"

Tier 2 — 长期影响（小时/天）
├── 持续暴雨 → 洪水风险
│   ├── 低洼地区被淹
│   ├── 农作物受损
│   └── 官府组织抗洪
├── 河流水位上涨 → 渡船可能停运 → 影响贸易路线
└── 生态：雨水充足 → 植物茂盛 → 来年丰收可能
```

### 2.4 级联示例——砍掉一棵古树

```
玩家砍伐古银杏

即时：
├── 获得：银杏木×20（上等材料）
├── 惊飞树上鸟群
└── 巨响引起注意

NPC 反应：
├── 村里老人：愤怒，好感-40
├── 以前在树下休息的人：不满，好感-10
├── 口碑传播："有人砍了古树"（传播范围大）
└── 可能被罚款 50-200文

环境：
├── 该区域鸟类-50%（失去栖息地）
├── 如靠近河岸，水土流失风险+30
└── 夏天这条路没有树荫了

不可逆：这棵树永远没了
```

## 三、因果链详解

### 3.1 天气因果链

| 天气状态 | 触发事件 | NPC反应 | 经济影响 |
|--------|----------|----------|----------|
| 小雨 | 行人避雨 | 户外NPC移向室内 | 伞销量↑ |
| 暴雨 | 街道冷清 | 几乎所有户外NPC回家 | 户外商品滞销 |
| 大暴雨 | 可能洪涝 | NPC恐慌 | 粮价暴涨 |
| 连续晴 | 丰收预期 | NPC心情好 | 粮价微跌 |
| 大雪 | 路面难行 | 出行减少 | 燃料涨价 |

**天气→NPC行为规则：**
- 暴雨时：户外NPC `go_home`; 商贩 `close_shop`; 茶馆NPC增多（避雨客涌入）
- 晴天时：正常行为; 街上NPC增多

### 3.2 经济因果链

```
L2农田产量 → 粮食供给 → 粮价
  供给 > 需求 × 1.2 → 粮价上涨
  供给 < 需求 × 0.8 → 粮价下跌

L2山林猎物 → 肉类供给 → 肉价
L2河流鱼获 → 鱼价
NPC交易行为 → 局部供需变化 → 短期价格波动
```

**经济→NPC行为规则：**
- 粮价 > 200文/石: NPC买不起粮，饥饿加速，可能偷窃/乞讨
- 布价低: NPC多买布，心情上升
- 热门商品短缺: 商贩囤货，玩家买不到

### 3.3 人口因果链

| 状态变化 | 触发事件 | 传播方式 |
|----------|----------|----------|
| 某区域人口变化>10% | 人口流动事件 | 茶馆传播 |
| 大量流民涌入 | 治安事件 | 官府反应 |
| 商人集中 | 市场繁荣 | 物价波动 |

### 3.4 生态因果链

```
狼群减少 → 兔子增多 → 菜地被啃 → 农田减产 → 粮价上涨
猎户增多 → 猎物减少 → 肉价上涨 → NPC抱怨
过度捕鱼 → 鱼群减少 → 渔夫收入下降 → 渔夫转行
```

### 3.5 NPC行为因果链

| NPC状态 | 触发行为 | 连锁效果 |
|--------|----------|----------|
| 饥饿<30 | 去买粮/乞讨/偷窃 | 粮铺生意好/治安事件 |
| 疲劳>80 | 去休息/回家 | 店铺暂时无人 |
| 铜钱<20 | 去打工/借钱/偷窃 | 码头劳力增加 |
| 铜钱>200 | 消费/进货/投资 | 市场繁荣 |
| 心情<30 | 去茶馆/找人聊天 | 茶馆生意好/传播消息 |

### 3.6 具体NPC因果链（10个核心NPC）

**王掌柜（布庄商贩）：**
- 布价低 → 进货（铜钱减少，库存增加）
- 有库存 → 开铺卖货（铜钱增加，库存减少）
- 库存=0 → "今天布卖完了" → 玩家买不到布
- 暴雨 → 关店 → 玩家进不了布庄
- 赚到钱(铜钱>200) → 去茶馆消费 → 传播商业消息

**刘寡妇（茶馆老板）：**
- 茶馆有客人 → 卖茶赚钱 → 传播最新消息（她是信息集散地）
- 知道某条消息 → 主动告诉玩家（"客官，您听说了吗？"）
- 天气差 → 客人少 → 收入减少 → 心情差
- 听到粮价上涨 → "最近日子不好过啊"

**码头管事：**
- 有船到港 → 分配搬运工作 → 玩家可以打工
- 没船 → "今天没活干" → 玩家不能打工
- 大船到港 → 搬运量大 → 工钱翻倍 → 消息传播"码头来了好大一条船"

**张三（混混/小偷）：**
- 饥饿<30且铜钱<10 → 偷窃（成功率基于附近NPC数量和衙役位置）
- 偷窃成功 → 被偷NPC铜钱减少 → 报官事件
- 被抓 → 关押 → 几回合不能行动 → 释放后声誉差
- 在茶馆 → 传播小道消息（不可靠信息，可能夸大）

**钱捕头（衙役）：**
- 张三偷窃 → 追捕 → 如果在同一Grid则立即抓
- 巡逻中 → 发现可疑行为 → 事件记录
- 城内治安好 → NPC心情上升
- 城内治安差 → NPC不敢出门 → 经济下滑

**李大夫（药铺）：**
- 有人生病 → 看病 → 收费 → 赚钱
- 药材充足 → 正常看病
- 药材短缺 → "药涨价了" → 无法看病 → 病情加重
- 城外有瘟疫 → 大量病人涌入 → 忙不过来 → 求助事件

**赵秀才（书生）：**
- 在贡院读书 → 才学增加
- 才学>阈值 → 可以参加科举 → 消息传播"听说下个月要加开一次恩科"
- 参加科举 → 中榜/落榜 → 命运改变
- 在茶馆 → 讨论时政/科举 → 传播消息

**陈猎户（猎户）：**
- 山里猎物多 → 打猎 → 获得猎物 → 肉价可能下降
- 山里猎物少 → "最近山里打不到东西" → 买不到肉
- 天气差 → 不进山 → 无猎物
- 受伤 → 去找李大夫 → 触发看病事件

**孙铁匠（铁匠）：**
- 有材料 → 锻造 → 产出铁器 → 卖掉赚钱
- 材料不足 → "最近矿石不好收" → 铁器涨价
- 暴雨 → 炉子灭 → 无法锻造
- 有大订单(如官府) → 加班 → 疲劳增加 → 需要休息

**周嫂子（农妇）：**
- 农田需要劳力 → 干活 → 产量增加
- 天气好 → 产量好 → 心情好 → 传播好消息
- 天气差 → 产量差 → "今年收成不好" → 粮价上涨消息
- 家人饿了 → 去买粮 → 触发经济交易

## 四、万物交互设计

### 4.1 交互层级

| 层级 | 操作类型 | 是否改变对象状态 | 示例 |
|------|----------|----------------|------|
| Level 0 | 被动感知 | 否 | 对象存在于世界中 |
| Level 1 | 观察（看/听/闻/摸） | 否 | "这是一棵百年银杏，叶已转黄" |
| Level 2 | 操作（拿/放/用/组合/破坏） | 是 | 折下柳枝→柳枝进背包+柳树状态变化 |

## 五、远方消息生成规则

### 5.1 远方消息必须有L2统计支撑

```
不能随机生成"陕西大旱"——除非L2统计中陕西区域确实干旱
不能随机生成"黄河决口"——除非L2统计中降雨量确实超标
不能随机生成"大船到港"——除非L2统计中确实有贸易船只到达
```

### 5.2 远方消息生成逻辑

```
每回合检查所有L2区域统计变化:
  for each region:
    if region.rainfall > threshold:
      generateEvent("flood_risk", region)
    if region.harvest < normal * 0.5:
      generateEvent("harvest_failure", region)
    if region.trade_volume > normal * 2:
      generateEvent("trade_boom", region)
    if region.animal_population < min_threshold:
      generateEvent("ecological_crisis", region)
```

### 5.3 消息传播规则

```
事件产生后:
1. 立即在发生区域可见
2. 1时辰后传播到相邻区域
3. 1天后传播到同城
4. 3天后通过商人传播到远方
5. 每次传播有20%概率细节丢失/夸大
```

## 六、NPC对话因果链

### 6.1 NPC只能说知道的事

```typescript
function generateNPCDialog(npc: NPC): string {
  // 检查NPC的记忆库
  const memories = npc.memory.recentEvents;

  if (memories.length === 0) {
    return null; // 这个NPC没什么可说的
  }

  // 根据NPC职业和性格筛选相关记忆
  const relevantMemories = memories.filter(m => isRelevant(npc.profession, m));

  if (relevantMemories.length === 0) {
    return null;
  }

  // 基于真实记忆生成对话
  return generateDialogFromMemory(npc, relevantMemories[0]);
}
```

### 6.2 对话模板基于事实

**❌ 错误：** "听说最近粮价要涨了"（随机）
**✅ 正确：**
- L2统计显示南郊农田产量下降30%
- → 经济系统：粮价上涨15%
- → 事件：粮价上涨
- → 传播到茶馆
- → 刘寡妇知道后说："最近南边遭了虫灾，今年新米怕是要涨价喽。"

**❌ 错误：** "码头来了好大一条船"（随机）
**✅ 正确：**
- L2统计显示河流区域贸易量增加200%
- → 事件：大型商船到港
- → 码头管事知道：分配20个搬运工
- → 传播到茶馆
- → 小孩跑过喊："快去看！码头来了好大一条船！"

## 七、叙述系统

### 7.1 设计规则

每个交互/事件对应一个叙述模板，包含多条文案，运行时加权随机选择。

**感知属性影响叙述丰富度**：

| 感知范围 | 示例 |
|----------|------|
| 0-30 | "一棵{树种}，{季节状态}。" |
| 30-60 | "这是一棵{树种}，{高度}，{健康状态}。{季节描述}。" |
| 60-100 | "这棵{树种}大约{年龄估计}，树干{细节}。{季节详细描述}" |

## 八、环境与生态系统

### 8.1 天气影响总览

| 天气 | NPC行为 | 物品 | 经济 | 生态 |
|------|---------|------|------|------|
| 晴天 | 正常 | 正常 | 正常 | 植物蒸发快 |
| 下雨 | 避雨/撑伞 | 露天物品受潮 | 伞涨/户外货跌 | 植物获得水分 |
| 暴雨 | 大部分回家 | 严重受潮 | 运输中断 | 可能洪涝 |
| 下雪 | 减少外出 | 水结冰 | 燃料涨价 | 植物休眠 |
| 大雾 | 行动谨慎 | 正常 | 渡船停运 | — |

### 8.2 生态食物链

```
高级捕食者：虎 → 鹿/野猪
中型捕食者：狼 → 兔/鸡 | 狐狸 → 兔/鸡
小型动物：兔 → 草 | 老鼠 → 谷物
昆虫类：蜜蜂 → 花蜜 | 蛐蛐 → 植物
水生类：大鱼 → 小鱼 → 虾/虫
```

**生态平衡规则**：
- 砍掉大树 → 鸟类减少 → 虫害增加 → 农作物受损
- 猎杀过多狼 → 兔子暴增 → 菜地被啃
- 过度捕鱼 → 鱼群减少 → 渔夫收入下降
- 保护生态是一种长期策略

### 8.3 动物种群动态（L2统计规则）

```
每日更新：
  当前数量 = 上日数量 × (1 + 繁殖率 - 狩猎压力 - 自然死亡)

  繁殖率：兔0.05, 鹿0.03, 野猪0.02, 狼0.01, 虎0.005
  自然死亡：0.01（所有种类）
  狩猎压力：取决于区域内猎人数 × 狩猎技能

  数量上限 = 区域容量上限
  数量 < 最低阈值 → "此区域动物稀少"
```

### 8.4 植物生长公式

```
每Tick生长量 = 基础生长率 × 品种因子 × 水分因子 × 养分因子 × 温度因子 × 健康因子

水分因子：
  moisture < 20: ×0.2（严重缺水）
  moisture 20-40: ×0.5
  moisture 40-80: ×1.0（适宜）
  moisture > 80: ×0.8（过多）
```
