# 17 — 组织系统（实体定义 + 成员归属 + 涌现逻辑）

## 一、设计目标

1. 新增组织实体类型，NPC 携带 `factionId` 明确归属
2. 组织属性参与事件涌现逻辑
3. 前端 NPC 详情面板显示组织信息，可跳转

## 二、实体定义

### 2.1 EntityType 扩展

```typescript
export enum EntityType {
  NPC = 'npc',
  ANIMAL = 'animal',
  PLANT = 'plant',
  MINERAL = 'mineral',
  BUILDING = 'building',
  ITEM = 'item',
  FACTION = 'faction',       // 统一用 FACTION，通过 type 字段区分子类型
}
```

### 2.2 FactionComponent

```typescript
export interface FactionComponent {
  name: string;              // 组织名称
  type: 'government' | 'military' | 'merchant' | 'religion' | 'underground' | 'scholar';
  influence: number;         // 影响力 0-100
  treasury: number;          // 金库（文）
  members: number[];         // 成员NPC ID列表
  leaderId: number;          // 首领NPC ID
  territory: string[];       // 控制的gridId列表
  relations: Record<number, number>; // 与其他组织的关系 (-100~100)
  goals: string[];           // 当前目标
  mood: number;              // 组织士气 0-100
}
```

### 2.3 IdentityComponent 扩展

```typescript
export interface IdentityComponent {
  name: string;
  profession: string;
  age: number;
  personality: string[];
  factionId?: number;              // 所属组织实体 ID（0 或 undefined = 无组织）
  factionRole?: 'leader' | 'member';  // 组织角色
}
```

### 2.5 初始化代码

在 `entities.ts` 中 `generateFactions()` 内：
- 创建组织实体，分配 leaderId/members
- 给每个 member NPC 的 Identity 组件写入 `factionId` 和 `factionRole`
- 首领 NPC 设 `factionRole: 'leader'`

## 三、API 与前端

### 3.1 后端查询

- `worldEngine.getNpcFaction(npcId)` → `{ id, name, role } | null`
- `worldEngine.getFactionMembersDetailed(factionId)` → 完整成员信息（含 profession, health, mood, gridId）

### 3.2 API 增强

- `GET /api/world/entity/:id` 响应中增加 `factionId` 和 `factionName` 字段
- `GET /api/world/factions/:id` 详情中成员信息更丰富（含 gridId, health, mood）

### 3.3 前端展示

- NPC 详情面板显示「所属组织」标签（含名称 + 角色）
- 点击组织名称跳转到组织详情面板
- 组织详情中点击成员跳转到 NPC 详情

### 4.3 emergenceRules.ts 新增规则

| 规则 | 效果 |
|------|------|
| 同组织认同 | 对同组织 NPC 好感度加成，交易/互动有折扣 |
| 跨组织冲突 | 敌对组织 NPC 互动可能触发冲突事件 |
| 组织任务 | 首领 NPC 可能发布组织相关任务 |
| 势力范围 | 在组织控制区域内，该组织 NPC 出现概率更高 |
| 组织资源 | 组织金库影响成员钱包（俸禄/分红） |
| 士气影响 | 组织 mood 低时，成员 mood 也受影响 |

## 五、数据流

```
NPC 创建 → generateFactions() 分配成员 → Identity.factionId 回写
                                                    ↓
entity/:id API → 返回 factionId + factionName → 前端展示
                                                    ↓
emergenceRules → 检查 ctx.target.identity.factionId → 触发组织相关涌现
```

## 六、开发优先级

1. **P0**: 新增 EntityType + FactionComponent + IdentityComponent 扩展
2. **P0**: 初始化 12 个组织实体 + 分配 leaderId/members + 回写 factionId
3. **P0**: API 增加组织字段 + 前端显示组织信息
4. **P1**: 组织涌现事件规则（emergenceRules.ts 新增 5-8 条）
5. **P1**: 组织间关系系统
6. **P2**: 组织状态每回合结算（influence/treasury/mood 变化）
